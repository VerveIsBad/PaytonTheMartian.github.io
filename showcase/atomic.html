Atomic

The Python modding platform.

Atomic is a modding API for Python, made to be fast and simple.

Installation

To include Atomic in your project, add the atomic folder to your project's root folder. Then copy the mods/ folder as well.

Next, add the following lines to your main file, or where your app/game gets initalized (Right before running the app/game):

from atomic import loader
loader.load_mods([
    'examplemod',
    # other mods to load.
])

Just like that, you've added Atomic.

Atomic will need some setup a well, in mods/basemod/, make some files and functions that can be called from other locations in your app/game. This way modders can hook into the Base Mod and in turn, mod your app/game.

Modding

The modding system of Atomic is intentionally small and simple.

To make a mod, there are only a few steps.

First, make a new folder in the mods folder. Make sure to use underscores rather than spaces. You can use a-Z, 0-9, and _. Note that the folder's name cannot start with a number.

Then, make a new __init__.py file inside your mod folder. This file gets executed when your mod is imported, so don't write much top-level code. Try to only use imports at the top-level.

Inside this __init__.py file, make three new functions. on_load(basemod), post_load(basemod, mods), and get_name().

on_load(basemod) is called when your mod is loaded, not all mods are loaded yet, so only basemod is provided in the args. This function has no return values.

post_load(basemod, mods) is called after all mods are loaded, so both basemod and all mods can be provided as arguments. This function also has no return values.

get_name() is called any time something needs to get the name of your mod. The return value of this function is the name of your mod.
